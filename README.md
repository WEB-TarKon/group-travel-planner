# group-travel-planner
## Проєкт дипломної роботи

Система для планування групових подорожей з маршрутами на карті, сповіщеннями та керуванням спільними витратами.

## Дипломні теми
- **Кондратенко Сергій:** Серверна частина забезпечення гарантійних внесків, поетапних виплат і боргових зобов’язань для планування групових подорожей
- **Тараненко Аліна:** Веб-інтерфейс і мобільний застосунок з картографічним маршрутом та сповіщеннями для планування групових подорожей

---

## Структура репозиторію
```
apps/
  backend/   # API (серверна частина)
  web/       # Веб-інтерфейс
  mobile/    # Мобільний застосунок (Expo)
  bot/       # Telegram-бот для сповіщень (опційно)
packages/
  shared/    # Спільні типи/DTO/enum (опційно)
```

## Вимоги

- **Git**
- **Node.js (LTS)**
- **Docker Desktop (для бази даних)**
- **WebStorm (або будь-яка IDE)**

## Швидкий старт (локально)

### 1) Клон репозиторію
```
git clone <REPO_URL>
cd group-travel-planner
```

### 2) Запуск бази даних (PostgreSQL) у Docker
Потрібен docker-compose.yml у корені репозиторію.
```
docker compose up -d
```

Перевірка:
```
docker ps
```

## Запуск застосунків

### Backend (API)
```
cd apps/backend
npm install
npm run start:dev
```

### Web (React)
```
cd apps/web
npm install
npm run dev
```

### Mobile (Expo)
```
cd apps/mobile
npm install
npx expo start
```

## Git workflow
### 1) main — тільки стабільна версія.
### 2) Кожна задача робиться в окремій гілці:

- feature/backend-...
- feature/web-...
- feature/mobile-...

### 3) Пушимо гілку → створюємо Pull Request → інший учасник переглядає → merge в main.

## Примітки
- Файли з секретами (.env) у репозиторій не додаємо.
- Всі змінні оточення та приклади конфігурації будуть описані окремо (наприклад, у .env.example).
